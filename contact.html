Next Steps

If given more time and resources, we would try to develop APP using node.js and set up the server for an IP camera, which will solve the delay issue, right now the delay varies because of the transmission layers of multiple protocols, and the cloud in Blynk is not as agile as we expect.

Regards to the robotic arm, the quality of the robot needs to be improved, and ideally it should move within a 360 degree hemisphere, right now due to the weight issue and quality problem, the movement is limited to a 2D surface, the corresponding Blynk joystick would have 4 direction to control.

Packing can also be improved by setting up a more durable outfit if given the access, the robot part needs to be robust to force acted by the cat, even if the cat jumps or try to grab it. The minor defect mentioned above sometimes show in testing, the next steps will be refining the project.
