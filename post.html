Security & Network

video streaming through RTSP

So we have considered streaming video as multimedia object, Proxies are commonly used to cache objects, and multimedia proxies are more likely to be exposed under unauthorized access, which increases of risk from the perspective of security . Real-time Streaming Protocol (RTSP) ranked better than other solutions to deal with latency and quality(loss).

The Real Time Streaming Protocol (RTSP) is a network control protocol has several benefit when I was doing my research, RTSP designed for VCR style command which is exactly we are looking for, in communications systems it is designed to control streaming media servers. Data transmission is also solved by this widely applied technology, the Real-time Transport Protocol (RTP) in conjunction with Real-time Control Protocol (RTCP) for media stream delivery.

In practice, we have initially used local host using motion library to prototype the surveillance camera, and it is locally hosted. Then we start to upload the streaming through a server port, but the webcam, unlike Raspberry Pi camera, it is not an IP camera and configuration is tough for RTSP set-up, several approaches are taken and we found the best solution is to use Android system to host the video streaming and Blynk accepts the streaming from cloud, then transmitted WIFI network and displayed on APP UI main page.
